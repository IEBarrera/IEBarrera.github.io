**moiréGL** is a program for synthesizing moiré patterns, as a vertical or
horizontal grid, from arbitrary 3D models, using their UV coordinates.

<!--more-->

## A simplified introduction

A moiré pattern appears when two similar patterns are superimposed. The
simplest case is the one formed by two grids of vertical (or horizontal)
stripes.

<canvas id="moire-grid" class="moire-live" aria-hidden="true"></canvas>
<script src="{{ '/assets/js/moire-grid.js' | relative_url }}" defer></script>

As you can see, superimposing two grids with a slight difference in the width of their stripes generates a new pattern. The reason is that, since their periods do not match, the relative position between the stripes changes gradually across the image: in some areas the stripes of one pattern fall over the gaps of the other and fill in the empty spaces, while in others the stripes coincide and leave the largest amount of empty space exposed. That periodic contrast between full and empty areas, with a period much larger than that of the original grids, is what produces the magnification effect, and it is the principle at work behind any moiré pattern.


## The idea

Synthesizing a moiré requires thinking the process in reverse: starting from the desired pattern and determining which superposition of grids produces it. Although the process is mathematically well described, it is possible to greatly simplify the problem by using a classic approach: grid animation.

To create a grid animation you need to apply an *interlacing* process to the desired animation, that is, joining all the frames in an interleaved fashion, cyclically taking one column from each frame at a time. In turn, the revealing grid must be built in such a way that it shows a single frame at a time:

A moiré pattern can be regarded as a traditional grid animation, where the animation is made up of a grid with the same proportions as the revealing grid, and where the revealing grid reveals one pixel of each frame at a time. This may seem counterintuitive; however, it is because what the pattern "reveals" is not the gap of the revealing grid itself, but the intersection between it and the lines of the deformed grid.

While this way of synthesizing is conceptually simple, at the computational level it is a waste of resources, since a large part of each animation frame is never used: you would have to render a large number of pixels only to discard more than you will actually use.
To avoid such waste, a question arises: is it possible to render only the pixels that are needed?

In practically any 3D engine it is possible to build materials (or shaders) that take into account different variables of the environment, such as the angle, the position or, what interests us here, the position in screen space. Being more technical, it is possible to bind the UV position of a grid texture on a 3D model to the NDC coordinates, which means that for every pixel on screen I can automatically select the animation "frame" of the grid on the 3D model.

With this approach moiréGL was created, using, as its name suggests, OpenGL. The result is an efficient synthesis in which the perspective projection of the 3D model deforms the grid through its UV coordinates, with depth only resolving occlusion, and in which the pattern is generated by applying a cyclic offset of the grid over those coordinates.

## Technical details

- **The grid is not a texture: it is evaluated per pixel in the fragment
  shader.** For each fragment, the model's UV coordinate is taken and an
  offset proportional to `mod(screen_x, spacing)` is added to it: each screen
  column within one period of the revealing grid "sees" a different phase of
  the pattern. It is classic interlacing, but solved in a single pass and
  without rendering a single pixel more than needed.
- **Native C++/OpenGL core** (`moire_native.pyd`, exposed to Python through
  pybind11): hidden GLFW context and *offscreen* rendering to an FBO —the
  resolution is only limited by the GPU—, OBJ loading with UVs, configurable
  camera and object transforms. Geometry, shader and framebuffer are cached
  between calls, which makes the interactive mode viable.
- **PyQt5 interface**: live controls for camera, geometry, line frequency and
  grid spacing, with auto-render at ~60 fps.
- **NumPy/Pillow compositing**: generates the revealing grid (antialiased via
  supersampling) and animates it (loop and ping-pong) by shifting it over the
  render.

## What can be created with this?

moiréGL was developed for the artist [Mario Morales Astudillo](https://www.instagram.com/mario.morales.astudillo), who brings moiré patterns into the real world: two plates, the interlaced image below and the revealing grid above, mounted at a precise separation, so that the pattern animates as the viewer moves in front of the work.
